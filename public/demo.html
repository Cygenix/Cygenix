Platform Features
Everything you need to
migrate with confidence
🔌
Live Database Connection
Connect directly to SQL Server on-premises or Azure SQL via connection string or Azure Function with Managed Identity. No VPN or firewall changes required.
🤖
AI Column Mapping
Claude AI automatically maps columns between source and target schemas, handling name variations, type differences, and suggesting transforms with confidence scores.
🔄
Was/Is Value Translation
Upload translation tables to automatically convert values during migration — status codes, lookup IDs, department names, currency codes, and more.
✅
1,500 Validation Scripts
Run pre and post-migration validation scripts directly from your database. Filterable by group, name, and database target. Results exported to Excel with problem rows.
📋
Conversion Projects
Sequence multiple table migrations and custom SQL scripts into a single ordered project. Retry, skip, or abort steps on failure. Full execution log.
📊
Professional Reports
Export client-ready Excel reports with summary, migration results, column mapping, Was/Is rules applied, and execution logs — ready to hand to stakeholders.
👤
Client Portal
Give clients their own login to a stripped-down portal for uploading source data, documents and notes. Files surface directly in the analyst's Inventory.
☁️
Azure Native
Built for Azure SQL. Managed Identity authentication means no passwords in connection strings. Deployed on Netlify with Azure Function backend.
📖
Inventory & Documentation
Store schema files, data dictionaries, mapping documents and Was/Is tables in an organised inventory. Preview CSVs, download documents, manage versions.
⚙️
Custom SQL & Stored Procedures
Inject any SQL script or stored procedure call into the migration sequence — run DELETE, TRUNCATE, EXEC, or complex transforms on source or target before, between, or after table migrations.

Custom SQL & Stored Procedures
Full SQL control at every step
Between table migrations, inject any SQL — bulk deletes, stored procedure calls, index rebuilds, data transforms, or complex business logic. Run on source or target.
  • Write any T-SQL — SELECT, UPDATE, DELETE, TRUNCATE, EXEC
  • Call stored procedures with parameters
  • Choose whether each script runs on source or target DB
  • Upload a .sql file or type directly in the editor
  • Errors captured in the log — retry, skip or abort
